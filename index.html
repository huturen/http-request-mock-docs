http-request-mock

Not only for mocking.

Quick Start →

Cross domain

Out of the box, with no configuration, support for cross domain.

Protobuf

Support for generating mock data by proto files.

Interceptor

It can be used as an interceptor. You can decide how to handle requests.

All in one

XMLHttpRequest, fetch, https.get, http.get, https.request, http.request.

More 3rd-party libraries

It supports axios, jquery, superagent, ky, node-fetch, got, you name it.

Dynamic mocking

Dynamically resolve response based on request headers, query and payloads.

Test capability

It has built-in support for unit test and works in jest and mocha environments.

Fake data

Easy to generate massive amounts of fake data.

Remote

Support for using remote mock data, where you can dynamically modify the data returned from the remote.

You can define your mock data like below:

/**
 * @url https://api.com/xyz
 * @method get
 * @header content-type: application/json
 * @header abc: xyz
 * @status 200
 * @delay 300
 */
const faker = require('http-request-mock/plugin/faker.js');
module.exports = (req) => {
  return {
    url: req.url,
    id: faker.incrementId(),
    enName: faker.name(),
    cnName: faker.name(true),
    age: faker.rand(10, 40),
    phone: faker.phone('(###) ###-####'),
   };
}

Introduction

It mocks http requests issued by axios, jquery, superagent, node-fetch, got, (… you name it) by intercepting XMLHttpRequest, fetch, and nodejs native HTTP/HTTPS module requests at the low level.

Note: The demos in this site do not compatible with IE11 and below.
